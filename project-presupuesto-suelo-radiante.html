Aplicación web para la gestión interna de artículos y elaboración de presupuestos para instalaciones de suelo radiante.

Proyecto

Análisis, planificación y desarrollo full-stack de una aplicación web para un cliente.

Descripción y contexto

El cliente necesitaba actualizar una aplicación de escritorio la cual estaba bastante limitada y ésta vez, quería que fuese con interfaz web para agilizar el proceso de los técnicos, que anteriormente debían escribir los datos de la vivienda para posteriormente, una vez en la central, introducirlos en la aplicación. Así también, los clientes con los conocimientos necesarios, podrían hacer sus presupuestos personalmente.
La aplicación constaría de dos partes diferenciadas: cálculo de presupuesto y panel de administración.

El cálculo del presupuesto cuenta con varias pantallas:


La parte izquierda con cabecera azul son los datos que el cliente/técnico ha de introducir para posteriomente hacer los cálculos para que se reflejen tanto en la parte derecha con cabecera negra como en la tabla de la parte inferior (totales de la instalación).
En la tabla de proyecto podemos añadir y eliminar estancias; calcular los datos, guardar la configuración actual descargando un fichero con extensión personalizada (aunque realmente es un JSON) y subir un fichero de configuración previamente guardado para cargar sus datos.

El siguiente paso una vez se realizan los cálculos, es pasar a la selección de artículos/componentes de la instalación.


Ésta pantalla cuenta con distintas pestañas para cada componente y dependiendo de modelos, tipos, variantes, sistemas, materiales y demás características de los componentes, las listas se irán actualizando para seleccionarlos.

Una vez se han seleccionado como mínimo los componentes imprescindibles, se podrá descargar un fichero PDF con el desglose de cada componente según metros, unidades, etc; y el total de la instalación.


Por motivos evidentes, no puedo poner capturas del panel de administración; pero se trata de un panel en el cual se pueden acceder a distintas categorías de componentes y datos (como parámetros de aislamiento, temperaturas según zonas geográficas...) para crear, eliminar o modificar. Dichos componentes se almacenan en una base de datos y cada uno cuenta con información básica como código, descripción, precio y adicionalmente, cada uno de ellos cuenta con sus datos más específicos como modelos, materiales, unidades por caja, metros por rollo, etc.
